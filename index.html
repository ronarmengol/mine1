Web developer vs web designer

Many people use the words web designer and web developer interchangeably. Let me add a little clarity on the difference.

The web designer specialises more on the look, design and the layout of the website and it's user experience. He uses his skills to attract, inform and persuade clients. He is adept in graphics software programs like Photoshop, where he enhances, manipulates images and text. The web developer on the other hand translates these designs into code which the computer can understand. He deploys the website into a hosting company and registers it with a search engine and optimizes the website for search engine search (SEO). The person who can design a webpage but cannot write or understand code is a graphic designer.

Your website must have a domain name. This is the reference point for your website in the internet. It is basically a web address. Domain names cost between $5 to $20 a year. Web hosting

Web hosting is the facility of hosting your website in internet. Local hosting charges vary, I have seen some starter rates at K230 per month, some K1400 if you pay for the full year.

You have many choices. If you want cheap rates they are out there. And you can upgrade once your business grows. Upgraded versions means serving the data faster.

One thing to keep your eye out is on the bandwidth the hosting company offers. They normally charge extra if you exceed the bandwidth for a particular hosting package. Let me illustrate, if you expect 20 visitors a day and let's say the average website page is 1.5MB (mega bytes - size of your website file stored in the hosting company) : in 30 days your bandwidth consumption will be : 20 visitors * 30 days * 1.5MB = 900MB. Normal start packages are 500MB. If you are starting, make sure you optimize your website and keep the number of images down.

How to make a website

First: you need a text editor. You can even start with Notepad, but do not use Word, programming uses different words and syntax and you will get tons of corrections. I use VScode. It is an integrated development environment (IDE). This is amazing for developers because it checks the code before compile time. There are others, like Atom, sublime, the choice is up to you.

The three languages you need to learn are HTML, CSS and a little Javascript. (not Java)

HTML is the chassis, the skeletal backbone, the content of the webpage. It is possible to make a webpage from HTML alone, but that would be pretty boring. CSS (cascading style sheets) this language as it suggests 'styles' your HTML elements. CSS has improved so much you can do some animation outside Javascript. Javascript provides program logic which HTML and CSS cannot do.

If you are new to programming, or new to HTML, CSS and javascript, you must practise and design until you become accustomed to the syntax (grammar) of the languages so the program works accordingly. Miss a comma, semi colon, and even a mispelling the program won't work.

Master your syntax, as you do this, and you will make tons of mistakes, from there you will learn basic debugging (finding program problems, and correcting them), and believe me, programs will get much more complicated.

Pro tip: do not memorize everything, most programmers cannot memorize the thousands of commands available. Learn how to use them and these tools as you need them.

Don't worry when you feel that you are spending more time staring at the screen than bashing in some code into your computer. You will spend considerable amount of time problem solving.

CSS has a large library of commands. I do not design layouts with CSS, it is time consuming and tricky. This is where Bootstrap reduces development time by incorporating code that is easy to implement. When you use Bootstrap your code design is cleaner and easier to manage.

Bootstrap is the most popular CSS framework. There are other CSS frameworks like Tailwind and postCSS.

Learn basic typography

Typography has to do with text design is presented on the webpage. The 'h1' tag is very important for SEO. Ensure one keyword is placed in an h1 tag. Use the 'h1' tag sparingly it should be used as a title for the webpage. Use the smaller headers for subtitles 'h2' to 'h6'. A common trend with titles is the lesser it becomes the smaller the font size. You can also use lower opacity to contrast from the greater title headers.

Use a minimum of two fonts on your page design. One for the titles and the other for the content.

Exercise care when you put text on images. Often times the text is difficult to read because of a low contrast value between the text and the image. In Google chrome tools you will score lowly on the accessibility score.

Do not use too many colours. Web designers use a maximum of 5 different palettes per page.

Make a provision also for your text to be read on mobile devices. Web designers suggest a 1.5 - 1.7 rem font size for content text.

When you prepare slabs of text for the bigger screens either use 2 - 3 columns for reading text. If you decide to use one columns reduce the max-width to 600 - 700px wide. It is common practise that text should not read the full width of a big screen.

Interrupt chunks of text with images, tables, charts, clippart, symbols, or the page will be boring and monotonous.

Website responsiveness

This is not about how fast the website is, but rather its responsiveness to different viewports like mobile devices. Developers create their websites on laptops or large screens, but they have to make designs as to how the website will look on a mobile device.

If the website is not mobile optimized it will take on two possible looks. One, the images and the text will shrink to fit into the mobile device, this will make the text and images too small to read and appreciate.

The second look: will be the images will be cropped and text will be overflowing on the horizontal axis. The user has to scroll left and right to read the text and see the complete image. This is not a pleasant user experience.

This can be solved by using a grid layout and media queries breakpoints. The responsive grid layout will force the content that is stacked horizontally to be stacked vertically. So there will be no need for the user to scroll left and right. Media queries, is a css function that can test the width of the viewport, if the viewport is the size of a mobile device, it can instruct the browser to load a mobile version of the image.

Website speed

This is obvious. No one likes a slow website.

Not only do we need to get into Google's good book, website studies around the world have shown that client's leave websites that take too long to load

Studies have shown that slow websites give an impression that it is not professionally made and the download speed is the first experience a visitor has over the website.

As a developer you are probably using at least 4G speeds to test and build your program, dont forget to take into account that the majority of people are using 3G on their mobile devices this means your program can take 3 times longer to download on their phones.

Some things you must resolve in program design to increase web speed:

  • defer render blocking scripts
  • remove unused scripts
  • put all scripts in one file to reduce redirects
  • use correct image formats
  • use correct image sizes
  • compress images
  • use media queries to select correct sized images for different screent sizes and aspect ratios
  • check for errors in the console
  • switch of auto play if you have videos
  • delay loading images that are under the fold
  • minify HTML, CSS and Javascript
  • Purge the CSS (do this cautiously)
  • prefetch 3rd party sources that will be used in later
  • minimize 3rd party downloads like imported fonts

SEO search engine optimization

SEO is the implementation of methods and design practises that follow search engine guidelines for a website to rank optimally. Organic search, is ranking in the search engine for particular keywords without the use of paid ads.

I have deliberately covered website responsiveness and website speed before SEO, because the two are important ranking factors for SEO design.

Google does not rank pretty websites. However, if you have an ugly website, it can reflect badly on your company image or profile.

Use images sparingly, and if you use them ensure you use the correct format and they are optimized. Images beautify your website, and they also increase file size. This means higher bandwidth and slower download times.
