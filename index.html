Website responsiveness

This is not about how fast the website is, but rather its responsiveness to different viewports like mobile devices. Developers create their websites on laptops or large screens, but they have to make designs as to how the website will look on a mobile device.

If the website is not mobile optimized it will take on two possible looks. One, the images and the text will shrink to fit into the mobile device, this will make the text and images too small to read and appreciate.

The second look: will be the images will be cropped and text will be overflowing on the horizontal axis. The user has to scroll left and right to read the text and see the complete image. This is not a pleasant user experience.

This can be solved by using a grid layout and media queries breakpoints. The responsive grid layout will force the content that is stacked horizontally to be stacked vertically. So there will be no need for the user to scroll left and right. Media queries, is a css function that can test the width of the viewport, if the viewport is the size of a mobile device, it can instruct the browser to load a mobile version of the image.

Website speed

This is obvious. No one likes a slow website.

Not only do we need to get into Google's good book, website studies around the world have shown that client's leave websites that take too long to load

Studies have shown that slow websites give an impression that it is not professionally made and the download speed is the first experience a visitor has over the website.

As a developer you are probably using at least 4G speeds to test and build your program, dont forget to take into account that the majority of people are using 3G on their mobile devices this means your program can take 3 times longer to download on their phones.

Some things you must resolve in program design to increase web speed:

  • defer render blocking scripts
  • remove unused scripts
  • put all scripts in one file to reduce redirects
  • use correct image formats
  • use correct image sizes
  • compress images
  • use media queries to select correct sized images for different screent sizes and aspect ratios
  • check for errors in the console
  • switch of auto play if you have videos
  • delay loading images that are under the fold
  • minify HTML, CSS and Javascript
  • Purge the CSS (do this cautiously)
  • prefetch 3rd party sources that will be used in later
  • minimize 3rd party downloads like imported fonts
